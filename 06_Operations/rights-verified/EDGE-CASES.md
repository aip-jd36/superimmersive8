# Rights Verified Edge Cases

**Version:** v0.1 (February 2026)
**Audience:** SI8 internal reviewer
**Purpose:** Growing log of gray areas, judgment calls, and precedents from real submissions

---

## How to Use This Document

When submission raises question not clearly answered by REVIEW-CRITERIA.md, document it here before making decision.

**This does two things:**
1. Forces clear thinking before judgment call
2. Creates searchable record for consistent future decisions

**When to create entry:**
- Submission contains something not covered by existing criteria
- Existing criteria ambiguous for this specific case
- About to make judgment call that could go either way
- Filmmaker challenges rejection with valid point

**When NOT to create entry:**
- Criteria is clear and case fits cleanly into Pass/Fail
- Issue is just missing documentation (request it, don't log it)

---

## Entry Format

```
## EC-[###] — [Date] — [Short descriptive title]

**Category:** [Which review category]
**Submission:** [Title / filmmaker — or anonymous]

**Situation:**
[What came up. Be specific. What did submission contain? What was ambiguous?]

**Decision:**
[What SI8 decided for this submission]

**Rationale:**
[Why this decision. What principles applied. What made it go this way vs. other.]

**Precedent:**
[How future similar cases should be handled. This is the reusable part.]

**Legal Review Needed:** [Yes / No / Flagged for v0.3]

**Status:** [Active precedent / Superseded by EC-###]
```
