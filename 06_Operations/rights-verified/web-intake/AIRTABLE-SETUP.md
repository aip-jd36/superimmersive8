## Step 4: Get Airtable API Credentials (3 min)

### **4A: Create Personal Access Token**

1. **Go to Personal Access Tokens page:**
   - Open: https://airtable.com/create/tokens
   - Or: Click your profile icon (top right) → "Developer hub" → "Personal access tokens"

2. **Create new token:**
   - Click "+ Create new token"
   - Token name: `SI8 Rights Verified API`
   - Description: `Web intake form API access`

3. **Add scopes:**
   - Under "Scopes", click "+ Add a scope"
   - Check these scopes:
     - ✅ `data.records:read` (read records)
     - ✅ `data.records:write` (create/update records)
     - ✅ `schema.bases:read` (read base schema)

4. **Add access to base:**
   - Under "Access", click "+ Add a base"
   - Select: `SI8 Rights Verified Submissions`
   - Click "Add base"

5. **Create token:**
   - Click "Create token"
   - **Copy the token immediately** (starts with `pat...`)
   - **IMPORTANT:** Save this token securely — you won't see it again
   - Example: `patABCDEFGHIJKLMNOPQRSTUVWXYZ1234567890`

6. **Store token safely:**
   - Paste into a secure text file (not in GitHub)
   - You'll need this for environment variables

---

### **4B: Get Base ID**

1. **Open your base:**
   - Go to https://airtable.com/
   - Open "SI8 Rights Verified Submissions" base

2. **Get Base ID from URL:**
   - Look at URL in browser
   - Format: `https://airtable.com/appXXXXXXXXXXXXXX/...`
   - Copy the part that starts with `app`
   - Example: `appABCDEFGHIJKLMN`
   - This is your **Base ID**

3. **Alternative method (via API documentation):**
   - Go to: https://airtable.com/developers/web/api/introduction
   - Click on your base name
   - You'll see "The ID of this base is appXXXXXXXXXXXXXX"

// Configure with your credentials
const AIRTABLE_API_KEY = 'patABCDEFGHIJKLMNOPQRSTUVWXYZ1234567890'; // Replace with your token
const AIRTABLE_BASE_ID = 'appABCDEFGHIJKLMN'; // Replace with your base ID

const airtable = new Airtable({ apiKey: AIRTABLE_API_KEY });

async function testConnection() {
  console.log('Testing Airtable connection...\n');

  // Test 1: Read table schema (check if base is accessible)
  try {
    console.log('📊 Testing base access...');

    const records = await base('Submissions')
      .select({ maxRecords: 1 })
      .firstPage();

    console.log(`   Found table: Submissions`);
    console.log(`   Current record count: ${records.length}\n`);
  } catch (error) {
    console.error('❌ Base access error:', error.message);
    console.error('   Check: API token has correct scopes and base access\n');
    return;
  }

  // Test 2: Create test record
  try {
    console.log('📝 Testing record creation...');

    const testRecord = await base('Submissions').create([
      {
        fields: {
          submission_id: 'TEST-2026-0001',
          status: 'received',
          filmmaker_name: 'Test Filmmaker',
          filmmaker_email: 'test@example.com',
          filmmaker_location: 'Test City',
          title: 'Test Film',
          runtime: '01:30',
          genre: 'Test Genre',
          first_submission: true,
          video_url: 'https://vimeo.com/test'
        }
      }
    ]);

    console.log('✅ Record creation works!');
    console.log(`   Created record ID: ${testRecord[0].id}`);
    console.log('   Check your Airtable base for the test record\n');

    // Test 3: Read the record we just created
    console.log('📖 Testing record read...');

    const readRecord = await base('Submissions').find(testRecord[0].id);

    console.log('✅ Record read works!');
    console.log(`   Read back: ${readRecord.get('filmmaker_name')}\n`);

    // Test 4: Delete test record (cleanup)
    console.log('🗑️  Cleaning up test record...');

    await base('Submissions').destroy([testRecord[0].id]);

    console.log('✅ Test record deleted\n');

  } catch (error) {
    console.error('❌ Record operation error:', error.message);
    if (error.statusCode === 422) {
      console.error('   Likely cause: Field name mismatch or missing required field');
      console.error('   Check: All field names match exactly (case-sensitive)\n');
    }
    return;
  }

  console.log('🎉 All tests passed! Airtable setup is complete.');
  console.log('\nNext step: Build the web form frontend');
}

testConnection();

## Troubleshooting

### **"Invalid API key" error**
- Make sure you copied the entire token (starts with `pat`, very long)
- Token is case-sensitive
- Make sure token has `data.records:read` and `data.records:write` scopes

### **"Could not find table" error**
- Table name is case-sensitive: must be exactly `Submissions` (capital S)
- Check table name at bottom of Airtable base

### **"Field X is not valid" error**
- Field names are case-sensitive: must match exactly
- Common mistake: `submission_id` vs `submissionId` (use underscore)
- Check field names in Airtable base match the schema exactly

### **"Permission denied" error**
- Make sure token has access to the correct base
- Go to token settings: https://airtable.com/create/tokens
- Click on your token → Check "Access" section → Should show "SI8 Rights Verified Submissions"

### **Rate limit error**
- Free tier: 5 requests per second per base
- Very unlikely to hit this during testing
- If hit, add delay between API calls

## Airtable Base Configuration Tips

### **Create useful views:**

1. **All Submissions** (Grid view, default)
   - Shows all records
   - Sort by `timestamp` (newest first)

2. **Pending Review** (Filtered view)
   - Filter: `status` = "received" OR "pre-screen" OR "full review"
   - Sort by `timestamp` (oldest first - review FIFO)

3. **Approved** (Filtered view)
   - Filter: `status` = "approved"
   - Shows only approved submissions

4. **Kanban Board** (for v0.2)
   - Group by: `status`
   - Drag-and-drop to change status

### **Set up automations (optional, v0.2):**
- Send email when status changes to "approved"
- Send email when status changes to "pending info"
- Slack notification when new submission arrives
